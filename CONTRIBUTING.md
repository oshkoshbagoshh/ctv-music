Contributing

Thank you for considering contributing to CTVMusic!

Getting started
- Fork and clone the repo.
- Create a virtualenv and install dependencies.
- Create a .env from .env.example.
- Run migrations and the dev server to verify your setup.

Development workflow
- Branch naming: feature/<short-description>, fix/<short-description>, docs/<short-description>
- Code style: follow the existing code style; keep imports ordered and code formatted. If you use linters/formatters, run them before commits.
- Tests: add or update tests for any behavior changes. Ensure `python manage.py test` passes locally.
- Commits: keep them small and descriptive.
- Pull requests: describe the changes and reference issues if applicable. Include screenshots for UI changes.

Running tests
- python manage.py test

Reporting issues
- Use GitHub issues. Include steps to reproduce, expected vs actual behavior, and environment details.

Security disclosures
- Please report security issues privately to the maintainer.
